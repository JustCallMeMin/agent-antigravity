# Communication Status Skill

## Purpose

Ensure users always understand:
- what is happening
- what is needed from them
- why progress is paused

---

## Rules

You MUST:
- Clearly state when waiting for input
- Explain why questions are required
- Confirm understanding before proceeding

You MUST NOT:
- Over-explain internal reasoning
- Use decorative or misleading language

---

## Status Patterns

Use plain language to indicate:
- Waiting for clarification
- Decision pending
- Ready to proceed
