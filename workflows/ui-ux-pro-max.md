## Invocation Rules

This workflow MAY be invoked only when:
- Designing or implementing complex UI or dashboards.
- Establishing or extending a design system.
- Performing high-fidelity UI/UX review or improvements.

This workflow MUST NOT be invoked when:
- Performing backend-only work.
- Fixing trivial UI bugs (use `/enhance`).

## Purpose

Design and implement premium UI/UX solutions using structured design systems,
validated patterns, and stack-specific best practices.

---

## Workflow Overview

When this workflow is triggered:

1. **Requirement Analysis**
   - Identify product type, industry, and target users.
   - Extract style and visual intent.
   - Detect target technology stack (default: html-tailwind).

2. **Design System Generation (Required)**
   - Generate a comprehensive design system using structured search.
   - Capture patterns, colors, typography, effects, and anti-patterns.

3. **Design System Persistence (Optional)**
   - Persist a master design system.
   - Create page-level overrides when required.

4. **Supplementary Research**
   - Perform focused searches for UX, typography, charts, or layout.

5. **Implementation Guidance**
   - Apply stack-specific best practices.
   - Ensure accessibility, responsiveness, and consistency.

---

## Output Expectations

The workflow MUST produce:
- A clear design system or design guidance.
- Stack-aligned UI recommendations.
- Explicit anti-patterns to avoid.
- Validation checklist before delivery.

---

## Key Principles

- Design systems precede components.
- Consistency outweighs novelty.
- Accessibility is mandatory.
- Visual quality must be verifiable.
